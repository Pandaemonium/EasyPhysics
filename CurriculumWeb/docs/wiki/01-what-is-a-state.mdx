# What Is a State?

Imagine pausing a chess game. If someone wanted to continue the game later, they would need to know where the pieces are and whose turn it is. They would not need to know what music was playing in the room.

That is the idea of a <Term name="state">state</Term>: the information a model needs right now in order to answer the question it was built for.

<ConceptFlow current="state" />

For a moving ball, a simple state might include:

- where the ball is
- how fast it is moving
- which direction it is moving

For a room, a useful state might include the temperature at many places. For a light wave, it might include the electric and magnetic field values. For a quantum system, the state becomes stranger and richer, but the basic role is the same: it tells the rule of change what it has to work with.

## State Is Not Everything

A state is not automatically a complete description of reality. It is an enough-now description for a purpose.

If you want to predict whether a thrown ball lands in a basket, you probably do not need to track the exact motion of every atom in the ball. If you want to predict whether the ball melts, those ignored details start to matter.

This is one of the quiet skills of physics: choosing the level of description that fits the question.
